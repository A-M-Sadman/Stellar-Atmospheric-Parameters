# Stellar-Atmospheric-Parameters
## AST402 Mid-Term Project — Stellar Atmospheric Parameters from Hβ Fitting

## Star analysed
- Name: tau Ceti
- HD number: HD 10700
- Archive: ELODIE (http://atlas.obs-hp.fr/elodie/)
- Data identifier: elodie:19970821/0032 (dataset 19970821, imanum 0032)

## Directory structure
- `data/observed/` — downloaded ELODIE spectrum (FITS)
- `data/phoenix_grid/` — downloaded PHOENIX-ACES model FITS files (108 spectra + 1 wavelength file)
- `src/` — all Python scripts (see execution order below)
- `figures/` — all report figures, generated by the scripts below
- `AST402_MidtermReport.docx` — final written report

## Setup
```bash
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

## Execution order

Run from inside `src/`, in this order:

1. `load_observed_spectrum.py` — Q1/Q2: load ELODIE spectrum, apply radial
   velocity + barycentric correction, continuum-normalize, produces Figures 1-2
2. `check_sigma.py` — Q2c: prints the exact per-pixel uncertainty value used
   in all later chi-squared calculations
3. `download_phoenix.py` — Q3a: downloads the 108-spectrum PHOENIX-ACES grid
   (takes a while; re-run to resume if interrupted, it skips completed files)
4. `model_loader.py` — Q3b: loads/convolves/normalizes a representative
   model, produces Figure 3
5. `plot_q4_comparisons.py` — Q4: Teff/log g/[Fe/H] comparison plots,
   produces Figures 4-6
6. `sersic_fit.py` — Q5/Q6: Sersic profile fit to the observed spectrum
   (Figure 7) and to the full 108-model grid (saves sersic_grid_fits.csv)
7. `chi2_grid.py` — Q7/Q8a: full chi-squared grid search, best-fit profile
   (Figure 8), delta-chi-squared projections (Figures 9a-c), parabolic
   refinement, formal uncertainty estimate (saves chi2_grid.csv)
8. `q8b_systematic.py` — Q8b: repeats the chi-squared search with continuum
   windows shifted +/-10 Angstrom to quantify systematic uncertainty
9. `q9_hr_diagram.py` — Q9: derives R, L from best-fit parameters and plots
   the H-R diagram (Figure 10)

`refit_grid.py` is a one-off utility used during development to re-fit the
grid after fixing 3 corrupted downloads; not part of the main pipeline.

## Known issues discovered and handled (see report Q3a, Q7-Q8 for discussion)
- PHOENIX archive uses 100 K native Teff spacing; 4 of the assignment's
  requested 250 K-step values were substituted with nearest valid grid points
- PHOENIX archive filenames use "-0.0" (not "+0.0") for solar metallicity
- log g and [Fe/H] are found to be unconstrained within the specified grid
  range by the chi-squared search (see Q7b/Q8a discussion)
